# Features

도메인 모델을 조합하여 사용자가 실제로 체감하는 구체적인 기능(유스케이스)을 구현하는 계층입니다. 도메인과 외부 라이브러리 사이의 어댑터 역할을 겸합니다.

## 폴더 구조 가이드 (각 기능 내부)

- `types.ts`: 도메인 모델에 라이브러리 전용 속성을 결합한 UI 특화 타입 정의
- `components/`: 특정 기능을 수행하기 위한 UI 컴포넌트 (라이브러리 종속적 컴포넌트 포함)
- `hooks/`: 사용자 액션 처리, 라이브러리 API 호출, 도메인 로직 연결 등을 수행하는 실행 로직
- `store/`: 해당 기능을 구현하기 위해 필요한 클라이언트 측 UI 상태 관리

## 작성 원칙

- 조합: 하나 이상의 도메인을 가져와 사용자의 목적(편집, 조회 등)에 맞게 조립합니다.
- 격리: 외부 라이브러리의 복잡한 설정이나 타입은 가급적 Feature 계층 안에서만 머물도록 합니다.
