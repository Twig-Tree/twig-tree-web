# Domains

서비스의 핵심 비즈니스 로직과 관련된 코드들을 도메인별로 묶어 응집도를 높이는 폴더입니다.
각 도메인은 독립적으로 존재하며, 다른 도메인에 대한 의존성을 최소화해야 합니다.

## 폴더 구조 가이드 (각 도메인 내부)

- `types.ts`: 도메인과 관련된 타입/인터페이스 정의
- `constants.ts`: 도메인 내에서 사용되는 상수 (초기값 설정 등)
- `utils/`: 도메인 데이터를 가공하거나 검사하는 순수 함수
- `hooks/`: 도메인 로직을 다루는 커스텀 훅 (ex: `useNodeContent`)

## 주요 도메인

- `tree/`: 트리 구조, 노드 생성/삭제, ELK 레이아웃 배치 등 '트리' 형태와 관련된 상태 및 로직
- `node/`: 트리를 구성하는 개별 '노드'의 상세 콘텐츠(태그, 메모, 텍스트)를 관리하는 로직
- `workspace/`: 사용자의 작업 공간 상태(현재 활성화된 탭, 에디터 모드 등)와 관련된 로직
- `user/`: 인증 정보 및 사용자 프로필 등 유저 관련 상태와 로직
