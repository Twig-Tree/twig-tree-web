# Domains

서비스의 본질적인 비즈니스 데이터와 규칙이 정의되는 계층입니다. 외부 환경(UI 프레임워크, 라이브러리)의 변화에 영향을 받지 않는 "순수성"을 유지합니다.

## 폴더 구조 가이드 (각 도메인 내부)

- `types.ts`: 시스템 전반에서 통용되는 핵심 데이터 모델 정의
- `constants.ts`: 비즈니스 로직의 기준이 되는 고정 값 및 초기 상태
- `utils/`: 데이터를 가공, 계산, 검증하는 순수 함수
- `hooks/`: 특정 기술에 종속되지 않고 비즈니스 데이터를 다루는 상태 로직

## 작성 원칙

- 독립성: UI 라이브러리(React Flow 등)의 타입을 직접 참조하지 않습니다.
- 응집도: 하나의 도메인 폴더는 자기 자신의 데이터 무결성에만 집중합니다.
